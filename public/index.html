LECAP / BONCAP Letras y Bonos Capitalizables

El instrumento mas simple. Compras a un precio, cobras un monto fijo al vencimiento. Sin cupones, sin ajuste.

Para el equipo de desarrollo

Precio → OMS tiempo real. En data912: ticker directo (ej: S17A6), campo c = ultimo precio.
Pago final → CNV fijo, de la licitacion del Ministerio de Economia. Se carga una vez, no cambia nunca.
Fecha de vencimiento → CNV fija, misma fuente.
Dias al vencimiento → CALC calculado: fecha_vto - settlement(hoy). No se hardcodea.

Que desarrollar: ① Endpoint que lea precio del OMS por ticker ② Tabla de instrumentos con pago_final y fecha_vto (se actualiza solo en nueva licitacion) ③ Calculo de TNA, TIR y Paridad — formulas directas, sin iteracion.
Es el mas simple. Si esto funciona, el 70% de la logica ya esta.

Formulas
ganancia = pago_final / precio
TNA = (ganancia - 1) × (365 / dias) × 100 // interes simple
TIR = (ganancia ^ (365 / dias) - 1) × 100 // interes compuesto
Paridad = precio / pago_final × 100

BONCER / LECER Ajustados por CER (inflacion)

El capital se ajusta por inflacion. Los flujos se multiplican por el coeficiente CER. La TIR es real (por encima de inflacion).

Para el equipo de desarrollo

Precio → OMS. En data912: TX26 (ojo: viene por 10 VN, dividir por 10).
CER actual → BCRA serie 30. Consultar 1 vez por dia y cachear.
CER de emision → CNV fijo por bono, del prospecto. TX26 = 22.544 (emitido 2021). LECERs nuevas tienen CER ~700 (base distinta, mismo calculo).
Cronograma de flujos → CNV fijo. Cada flujo: fecha, % amortizacion, tasa cupon, base (0.5 = semestral).

Que desarrollar: ① Job diario que consulte CER a BCRA serie 30 y lo persista ② Para cada flujo futuro: interes = VR × tasa × base, ajustar por coef_CER ③ Newton-Raphson sobre flujos ajustados ④ Paridad = precio / (VR × coef_CER × 100).
Complejidad media. Las LECER son mas simples (pago unico × coef_CER, sin cupones intermedios).

Formulas
coef_CER = CER_actual / CER_emision
VR(i) = 1 - Σ amortizaciones anteriores
interes = VR × tasa_cupon × base
flujo_ajustado = (interes + amortizacion) × coef_CER
TIR → Newton-Raphson: precio/100 = Σ[flujo(i) / (1+r)^t(i)]

BONAR / GLOBAL Soberanos en USD

Bonos soberanos en dolares. Ley local (AL = Bonar) vs Ley NY (GD = Global). Mismos flujos, distinta jurisdiccion. Precio en USD.

Para el equipo de desarrollo

Precio en USD → OMS. En data912: usar ticker con sufijo D (MEP). Ej: AL30D = 61.13 USD. Sin sufijo es en ARS.
Precio en ARS → Para CCL implicito: CCL = AL30 (ARS) / AL30D (USD).
Flujos en USD → CNV fijos, del prospecto. Cupon + amortizacion por 100 VN. AL30 y GD30 tienen mismos flujos.

Que desarrollar: ① Mismo Newton-Raphson que BOTE pero en USD ② Comparar TIR local vs NY → spread = prima jurisdiccional ③ CCL implicito desde par ARS/USD ④ Duration.
Mismo calculo que BOTE pero en dolares. Complejidad en las unidades de data912 (miles para ARS, directo para USD).

BOTE Bonos a Tasa Fija con Cupon

Cupon fijo periodico + amortizacion. Sin ajuste. Todos los flujos son conocidos de antemano. El caso mas limpio de Newton-Raphson.

Para el equipo de desarrollo

Precio → OMS. En data912: ticker directo (TTS26), campo c.
Cronograma de flujos → CNV fijo. Cada flujo es un monto fijo en pesos (cupon + amort). El ultimo incluye devolucion de capital.

Que desarrollar: ① Newton-Raphson para TIR (misma funcion que soberanos/BONCER) ② Filtrar solo flujos futuros ③ Macaulay Duration.
El caso mas limpio de Newton-Raphson. Sin coeficientes, sin ajustes. Ideal para validar tu implementacion antes de pasar a CER/TAMAR.

Formula — Newton-Raphson
// Encontrar r (TIR) tal que:
precio/100 = Σ [ flujo(i) / (1 + r)^t(i) ]

// Iteracion:
r_nuevo = r - (PV - precio) / dPV
// Converge cuando |PV - precio| < 0.0001

function calcYTM(price, flows, settlement) {
  const MS_YR = 365.25 * 24 * 3600 * 1000;
  let r = 0.10;
  for (let i = 0; i < 100; i++) {
    let pv = 0, dpv = 0;
    for (const f of flows) {
      const t = (f.fecha - settlement) / MS_YR;
      if (t <= 0) continue;
      const d = Math.pow(1 + r, t);
      pv += f.monto / d;
      dpv -= t * f.monto / (d * (1 + r));
    }
    if (Math.abs(pv - price) < 0.0001) break;
    r -= (pv - price) / dpv;
    r = Math.max(-0.5, Math.min(2, r));
  }
  return r * 100;
}

LELINK Letras Dolar Linked

El capital se ajusta por el tipo de cambio oficial (A3500). Cobertura contra devaluacion.

Para el equipo de desarrollo

Precio → OMS. En data912: D30A6 (ojo: viene en miles de ARS, dividir por 1000).
TC Oficial actual → BCRA serie 4 (A3500). Consulta diaria.
TC de emision → CNV fijo, del prospecto de licitacion.

Que desarrollar: ① Job diario para TC Oficial BCRA serie 4 ② Coef DL = TC_actual / TC_emision ③ TIR y TNA en pesos (incluyen devaluacion) ④ Paridad contra VT ajustado.
Mismo nivel que LECER pero con tipo de cambio en vez de CER. La TIR es en pesos — para USD restarle la devaluacion esperada.

Formulas
coef_DL = TC_actual / TC_emision
pago_ajustado = pago_final × coef_DL
TIR (ARS) = (pago_ajustado / precio)^(365/dias) - 1
Paridad = precio / (100 × coef_DL) × 100
La TIR es en pesos. Para ver rendimiento en USD: TIR_USD ≈ TIR_ARS - tasa de devaluacion.

LETAM Letras ajustadas por TAMAR

Ajustado por TAMAR — la tasa mayorista de plazos fijos. BCRA publica la tasa diaria, nosotros la acumulamos.

Para el equipo de desarrollo

Precio → OMS. En data912: M31G6, campo c.
TAMAR actual → BCRA serie 44 (privados, TNA). Hoy: 26.75%. Consulta diaria.
Serie historica de TAMAR → BCRA serie 44 COMPLETA desde fecha de emision. Se necesita toda para acumular.
Fecha de emision → CNV fija, del prospecto.

Que desarrollar: ① Job diario que consulte TAMAR serie 44 y la persista en DB ② Funcion que acumule: coef = Π(1 + tasa/100/365) dia por dia desde emision ③ Proyectar TAMAR constante para flujos futuros ④ TIR estimada (siempre con disclaimer).
El mas complejo en datos. A diferencia de CER (que BCRA publica acumulado y solo dividis), TAMAR hay que acumularla dia por dia. Necesitas persistir la serie historica.

Formulas
// TAMAR se acumula DIARIAMENTE (distinto a CER que viene acumulado)
coef_TAMAR = Π [ 1 + TAMAR_dia(i) / 365 ] // para cada dia desde emision

pago_estimado = 100 × coef_TAMAR_total
TIR (est) = (pago_estimado / precio)^(365/dias) - 1
La TIR es siempre ESTIMADA porque depende de la TAMAR futura (desconocida). Se asume TAMAR constante.

BONTAM Bonos ajustados por TAMAR

El mas complejo. Combina cupones periodicos (TAMAR + spread), amortizacion de capital, y tasa variable.

Para el equipo de desarrollo

Precio → OMS. En data912: TMF27, campo c.
TAMAR + serie historica → BCRA serie 44. Misma que LETAM.
Spread sobre TAMAR → CNV fijo, del prospecto. TMF27 = 0%.
Cronograma de flujos → CNV fijo. Cada flujo: fecha, % amort, base. No tiene tasa fija porque la tasa ES la TAMAR.

Que desarrollar: ① Todo lo de LETAM (serie TAMAR, acumulacion) ② Cupon = VR × (TAMAR_periodo + spread) × base ③ Newton-Raphson sobre flujos proyectados ④ TIR siempre estimada.
El mas complejo de todos. Combina Newton-Raphson (como BONCER) con TAMAR acumulada (como LETAM). Dejarlo para el final.

Formulas
cupon = VR × (TAMAR_acum + spread) × base
flujo(i) = cupon + amortizacion
TIR (est) → Newton-Raphson sobre flujos proyectados
